## üìã CONTENT AUDIT FRAMEWORK

### The 7-Point Quality Assessment

**For each topic, evaluate these criteria:**

#### 1. Title Quality (0-10 points)
```
10 points: Descriptive, keyword-rich, clear intent
Example: "Complete Guide to Everest Base Camp Trek: Permits, Costs & Best Season 2026"

5 points: Generic but understandable
Example: "Everest Base Camp Information"

0 points: Vague or clickbait
Example: "Help needed" or "Question about trek"
```

#### 2. Opening Post Depth (0-10 points)
```
10 points: 300+ words, comprehensive, structured
8 points: 200-299 words, good detail
5 points: 100-199 words, basic info
3 points: 50-99 words, minimal
0 points: <50 words, one-liner
```

#### 3. Discussion Quality (0-10 points)
```
10 points: 10+ substantial replies, ongoing discussion
8 points: 5-9 quality replies
5 points: 3-4 replies
3 points: 1-2 short replies
0 points: No replies or only "thanks"
```

#### 4. Engagement Metrics (0-10 points)
```
10 points: 500+ views, multiple likes, shares
8 points: 200-499 views, some engagement
5 points: 100-199 views
3 points: 50-99 views
0 points: <50 views
```

#### 5. Uniqueness (0-10 points)
```
10 points: Original perspective, unique insights
8 points: Good information, somewhat unique
5 points: Standard information
3 points: Repetitive of other topics
0 points: Duplicate content
```

#### 6. Completeness (0-10 points)
```
10 points: Question fully answered, solution provided
8 points: Good answers, mostly resolved
5 points: Partial answer
3 points: Unanswered or unclear
0 points: No resolution, abandoned
```

#### 7. Current Relevance (0-10 points)
```
10 points: Recent topic or updated info
8 points: Still relevant, recent activity
5 points: Dated but still useful
3 points: Somewhat outdated
0 points: Completely outdated or obsolete
```

### Overall Quality Score
```
Total Points: __/70

60-70: Excellent - Minor tweaks only
50-59: Good - Moderate improvements
40-49: Fair - Significant work needed
30-39: Poor - Major overhaul or merge
0-29: Very Poor - Noindex or delete
```

## üîÑ MERGING DUPLICATE TOPICS

### When to Merge

**Merge if topics discuss:**
- Same subject with similar questions
- Complementary information that fits together
- One is clearly better quality
- Combined would create comprehensive resource

### How to Merge Effectively

#### Step 1: Identify Best "Anchor" Topic
```
Choose the topic with:
‚úì Best title
‚úì Most engagement
‚úì Highest quality opening post
‚úì Most replies
‚úì Best URL structure

This becomes your destination topic.
```

#### Step 2: Improve Anchor Topic First
```
1. Optimize the title
2. Enhance opening post
3. Add information from other topics
4. Create comprehensive resource
5. Add structure/sections
```

#### Step 3: Move Posts from Duplicates
```
In Discourse:
1. Open duplicate topic
2. Select valuable posts (skip "thanks" posts)
3. Click wrench icon ‚Üí Move posts
4. Choose destination topic
5. Move posts
6. Repeat for all duplicates
```

#### Step 4: Handle Empty Topics
```
For now-empty duplicate topics:
1. Edit first post to add redirect message:
   "This topic has been merged with [Link to anchor topic]"
2. Close the topic
3. Or set up redirect (Admin ‚Üí Permalinks)
```

#### Step 5: Add Merge Note
```
In the anchor topic, add:
"**Note:** This topic consolidates discussions from multiple related threads for easier reference."
```

### Merge Example

**Before: 3 Separate Topics**
```
Topic A: "Everest trek permit" (5 replies, 120 views)
Topic B: "How to get trekking permits" (3 replies, 80 views)
Topic C: "Permit costs for Everest" (2 replies, 45 views)
```

**After: 1 Comprehensive Topic**
```
Topic: "Complete Guide to Everest Trekking Permits: Costs, Process & Requirements 2026"

Content from all 3 topics merged
Total: 10 valuable posts
Combined: 245+ views
Result: One authoritative resource instead of 3 thin topics
```

## üö´ NOINDEXING LOW-VALUE CONTENT

### When to Noindex

**Noindex topics that:**
- Are thin and can't be improved (1-2 short posts)
- Are outdated and no longer relevant
- Are duplicates you don't want to merge
- Have very low engagement (<20 views after 6+ months)
- Are spam/low quality but kept for historical reasons
- Serve community but not search visitors

### How to Noindex (Admin Methods)

#### Method 1: Move to Noindex Category
```
1. Admin ‚Üí Categories ‚Üí New Category
2. Name: "Archived Topics" or "Historical"
3. Settings:
   - Include in search: UNCHECKED
   - Visible: Set appropriately
4. Bulk move low-value topics here
5. Topics stay for users, hidden from Google
```

#### Method 2: robots.txt Patterns
```
Admin ‚Üí Customize ‚Üí robots.txt

Add patterns for low-value sections:
Disallow: /c/archived/
Disallow: /tag/outdated/
Disallow: /tag/spam/
```

#### Method 3: Close Topics
```
Closed topics may get deprioritized by Google.
Not guaranteed, but worth trying for borderline cases.
```
